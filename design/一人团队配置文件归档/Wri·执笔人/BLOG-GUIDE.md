## 一、Front Matter 格式

```yaml
---
title: 文章标题
description: 文章描述（可选，100-200 字）
categories: [主分类，子分类]
tags: [标签 1, 标签 2, 标签 3]
date: 2026-03-09 12:00:00 +0800
draft: true  # 待评审时设置为 true
---
```

### 字段说明

| 字段 | 必填 | 说明 | 示例 |
|------|------|------|------|
| `title` | ✅ | 文章标题 | `Go 协程入门：从原理到实践` |
| `description` | ⚠️ | 文章描述（SEO 优化） | `OpenClaw 实战手记，持续更新实践和技巧` |
| `categories` | ✅ | 分类（数组格式） | `[AI, AI 能力集]` 或 `[编程语言，Go]` |
| `tags` | ✅ | 标签（数组格式） | `[AI, OpenClaw]` |
| `date` | ✅ | 发布日期（+0800 时区） | `2026-03-09 12:00:00 +0800` |
| `draft` | ⚠️ | 草稿标记（评审时为 true） | `true` / `false` |

## 三、写作规范

### 3.1 标题层级

```
# 一级标题（文章标题，只能用一次）
## 二级标题（主要章节）
### 三级标题（子章节）
#### 四级标题（如需要，尽量少用）
```

### 3.2 代码块

**必须指定语言**：
````markdown
```go      // ✅ 正确
```python  // ✅ 正确
```bash   // ✅ 正确
```       // ❌ 避免
````

**代码块要有说明**：
```markdown
**示例代码**：
```go
func main() {
    // 代码内容
}
```

**输出结果**：
```
Hello World
```
```

### 3.3 图片引用

**格式**：
```markdown
![图片描述](/images/文件名.png)
```

**图片说明**：
```markdown
![架构图](/images/2026-03-09-architecture.png)
*图 1：系统架构图*
```

### 3.4 表格规范

```markdown
| 列 1 | 列 2 | 列 3 |
|------|------|------|
| 内容 | 内容 | 内容 |
```

### 3.5 引用块

```markdown
> 重要提示、注意事项用引用块

> 环境说明：安装在家里单独的主机
```

### 3.6 链接格式

```markdown
[链接文本](完整 URL)
```

**示例**：
```markdown
[OpenClaw 官方文档](https://docs.openclaw.ai)
```

## 五、发布流程

### 5.1 写作阶段

```
1. Wri·执笔人 撰写博客
   ↓
2. 设置 draft: true
   ↓
3. 小黑 - 管家 提交评审
   ↓
4. 用户确认/修改
   ↓
5. 设置 draft: false
   ↓
6. Git 提交发布
```

### 5.2 评审消息格式

```
🖤 小黑：博客已写好，请老板评审！

【博客预览】
标题：{标题}
路径：{_posts/文件名.md}
GitHub: {预览链接}

【内容摘要】
- {要点 1}
- {要点 2}

请回复：
- "通过" → 设置 draft: false 并提交
- "修改 XXX" → 修改后重新评审
```

## 六、图片优化规范

### 6.1 图片格式选择

| 场景 | 推荐格式 | 说明 |
|------|---------|------|
| 截图/图表 | PNG → WebP | 无损压缩，体积更小 |
| 照片 | JPEG → WebP | 有损压缩，适合照片 |
| 图标/Logo | SVG | 矢量图，无限缩放 |
| 动图 | GIF → WebP | WebP 支持动图，体积更小 |

### 6.2 图片尺寸规范

| 用途 | 推荐宽度 | 说明 |
|------|---------|------|
| 正文插图 | 800-1200px | 适配大多数屏幕 |
| 缩略图 | 400px | 列表页使用 |
| 全屏图 | 1920px | 最多不超过此宽度 |

### 6.3 文件大小限制

| 类型 | 目标大小 | 警告线 |
|------|---------|--------|
| 普通截图 | < 200KB | > 500KB 需优化 |
| 复杂图表 | < 500KB | > 1MB 需优化 |
| 照片 | < 300KB | > 800KB 需优化 |

## 七、SEO 优化

### 7.1 标题优化

- ✅ 包含关键词
- ✅ 长度 20-60 字
- ✅ 避免标题党

### 7.2 描述优化

- ✅ 100-200 字概括全文
- ✅ 包含核心关键词
- ✅ 吸引点击

### 7.3 标签优化

- ✅ 3-5 个标签
- ✅ 包含技术栈 + 领域
- ✅ 避免过多标签
