## 2. 核心架构

### 2.1 整体架构图

```
┌─────────────────────────────────────────────────────────┐
│                    用户界面层                              │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐              │
│  │  飞书    │  │  微信    │  │  Telegram│  ...         │
│  └──────────┘  └──────────┘  └──────────┘              │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                   Gateway 网关层                          │
│  ┌─────────────────────────────────────────────────┐   │
│  │  消息路由 | 认证授权 | 插件系统 | 会话管理       │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                    Agent 代理层                           │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐              │
│  │  Main    │  │  Sub     │  │  ACP     │              │
│  │  Agent   │  │  Agents  │  │  Harness │              │
│  └──────────┘  └──────────┘  └──────────┘              │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                    工具执行层                             │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐              │
│  │  文件    │  │  网络    │  │  消息    │  ...         │
│  │  操作    │  │  搜索    │  │  发送    │              │
│  └──────────┘  └──────────┘  └──────────┘              │
└─────────────────────────────────────────────────────────┘
```

### 2.2 核心组件

| 组件 | 位置 | 作用 |
|------|------|------|
| **Gateway** | 后台服务 | 消息网关、认证、插件加载 |
| **Agents** | `/root/.openclaw/agents/` | AI 代理配置和会话 |
| **Workspace** | `/root/.openclaw/workspace/` | 工作区文件 |
| **Extensions** | `/root/.openclaw/extensions/` | 插件/技能 |
| **Config** | `/root/.openclaw/openclaw.json` | 主配置 |

### 2.3 数据流向

```
用户消息 → Channel Plugin → Gateway → Agent → Tools → 响应
     ↑                                                        ↓
     └─────────────────── 消息返回 ───────────────────────────┘
```

## 3. 配置详解

### 3.1 配置文件层级

```
优先级：高 → 低

1. 会话级配置 (session config)
2. 项目级配置 (.openclaw/ 内)
3. 用户级配置 (~/.openclaw/openclaw.json)
4. 系统级配置 (全局默认)
```

### 3.2 关键配置项

#### 3.2.1 模型配置

```json
{
  "models": {
    "mode": "merge",
    "providers": {
      "bailian": {
        "baseUrl": "https://...",
        "apiKey": "sk-...",
        "api": "openai-completions",
        "models": [
          {
            "id": "qwen3.5-plus",
            "contextWindow": 1000000,
            "maxTokens": 65536
          }
        ]
      }
    }
  }
}
```

#### 3.2.2 渠道配置

```json
{
  "channels": {
    "feishu": {
      "enabled": true,
      "appId": "cli_xxx",
      "appSecret": "xxx",
      "connectionMode": "websocket",
      "groupPolicy": "open",
      "dmPolicy": "open"
    }
  }
}
```

#### 3.2.3 工具配置

```json
{
  "tools": {
    "profile": "full"  // full | limited | minimal
  }
}
```

#### 3.2.4 Gateway 配置

```json
{
  "gateway": {
    "port": 18789,
    "mode": "local",
    "bind": "loopback",
    "auth": {
      "mode": "token",
      "token": "your-token"
    }
  }
}
```

## 4. 工具系统

### 4.1 内置工具分类

| 类别 | 工具 | 描述 |
|------|------|------|
| **文件** | `read`, `write`, `edit` | 文件读写编辑 |
| **执行** | `exec`, `process` | 命令执行和管理 |
| **网络** | `web_search`, `web_fetch` | 搜索和抓取 |
| **消息** | `message` | 发送消息 |
| **会话** | `sessions_*` | 会话管理 |
| **Feishu** | `feishu_*` | 飞书集成 |
| **其他** | `browser`, `canvas`, `nodes` | 高级功能 |

### 4.2 工具调用示例

#### 4.2.1 文件操作

```markdown
# 读取文件
read(path="/path/to/file.txt", limit=100)

# 写入文件
write(path="/path/to/file.txt", content="Hello World")

# 精确编辑
edit(path="/path/to/file.txt", 
     oldText="original text", 
     newText="new text")
```

#### 4.2.2 命令执行

```markdown
# 简单命令
exec(command="ls -la", yieldMs=5000)

# 后台命令
exec(command="npm run dev", background=true)

# 管理进程
process(action="list")
process(action="poll", sessionId="xxx", timeout=10000)
```

#### 4.2.3 网络操作

```markdown
# 搜索
web_search(query="AI news", count=10, freshness="pd")

# 抓取
web_fetch(url="https://example.com", extractMode="markdown")
```

### 4.3 工具使用最佳实践

```markdown
✅ DO:
- 使用 read 前先确认文件存在
- 大文件用 offset/limit 分块读取
- 敏感操作前询问用户
- 后台命令设置合理 yieldMs

❌ DON'T:
- 直接 rm -rf 危险目录
- 无限制读取大文件
- 在群聊泄露私人数据
- 轮询进程造成资源浪费
```

## 5. 技能开发

### 5.1 什么是技能 (Skill)

技能是 OpenClaw 的**可扩展功能模块**，包含：
- `SKILL.md` - 技能说明和激活条件
- 脚本/代码 - 实现逻辑
- 资源文件 - 配置、模板等

### 5.2 技能目录结构

```
skills/
└── my-skill/
    ├── SKILL.md          # 技能描述（必需）
    ├── index.ts          # 主逻辑
    ├── utils/            # 工具函数
    ├── templates/        # 模板文件
    └── assets/           # 资源文件
```

### 5.3 创建技能步骤

#### Step 1: 创建 SKILL.md

```markdown
# My Skill

## 描述
简要说明技能用途

## 激活条件
什么情况下触发此技能

## 使用方法
如何使用此技能

## 示例
使用示例
```

#### Step 2: 实现逻辑

```typescript
// index.ts
export async function run(params: any) {
  // 你的逻辑
  return { result: "success" };
}
```

#### Step 3: 注册技能

在配置中添加：
```json
{
  "skills": {
    "my-skill": {
      "enabled": true,
      "path": "/path/to/skill"
    }
  }
}
```
