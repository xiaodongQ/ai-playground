# AI 领取任务系统设计文档

> 原始设计日期: 2026-04-17

## 1. 项目概述

**项目名称**: AI Task System
**项目类型**: 个人 AI 任务管理与执行平台
**核心功能**: 可视化任务池 + 自动领取执行 + 交叉评估 + 迭代闭环
**目标用户**: 个人开发者，使用腾讯 Code Buddy IDE/CLI

## 2. 设计决策

### 2.1 为什么选择这些技术？

| 决策 | 选择 | 原因 |
|------|------|------|
| 后端框架 | FastAPI | 轻量、异步支持好、自动生成 API 文档 |
| 数据库 | SQLite | 本地存储、零配置、足够个人使用 |
| 前端 | 纯 HTML + Tailwind | 无需构建、最简部署、单文件可维护 |
| 执行方式 | Code Buddy CLI | 用户已有工具、直接复用 |
| 模型交互 | Markdown | 结构化、人类可读、便于调试 |

### 2.2 核心设计理念

1. **本地优先** - 所有数据存储在本地，不依赖云服务
2. **简单可靠** - 最小依赖、最少服务组件
3. **可观测性** - 完整记录执行过程，便于追踪问题
4. **迭代闭环** - 评估结果自动反馈，形成持续改进

```
┌──────────────────────────────────────────────────────────────┐
│                                                              │
│   pending ──▶ running ──▶ completed ──▶ evaluating ──▶ evaluated
│       │           │            │               │              │
│       │           │            │               ▼              │
│       │           │            │          re-execute ───────┘
│       │           │            │               │
│       │           │            └────── (评分<阈值) ───────────┘
│       │           │
│       │           └─────────────────────────────── (评分>=阈值)
│       │
│       └───────────────────────────────────────────────────── (人工删除)
└──────────────────────────────────────────────────────────────┘
```

## 6. 执行流程

### 6.1 任务创建

1. 用户在 Web UI 填写任务信息
2. 选择执行模型和评估模型
3. 任务入库，状态为 `pending`

### 6.2 任务执行

1. 调度器轮询 `pending` 状态任务
2. 领取任务，状态变为 `running`
3. 构建 CLI 命令并执行：
   ```bash
   claw --task-id <id> --model <model> "<description>"
   ```
4. 捕获执行输出，状态变为 `completed`
5. 自动触发评估

### 6.3 评估流程

1. 状态变为 `evaluating`
2. 构建评估 Markdown：
   ```markdown
   ## 任务
   <任务描述>

   ## 执行结果
   <执行输出>

   ## 评估报告
   ### 评分: X/10
   ### 优点
   - ...
   ### 问题
   - ...
   ### 改进建议
   1. ...
   ```
3. 调用评估模型 API 获取评估结果
4. 解析评分，低于阈值则触发重试

### 6.4 迭代闭环

评估结果以 Markdown 格式反馈给执行引擎：
```markdown
## 反馈给执行引擎
基于评估报告的改进建议，请重新执行任务。这是第 N 次迭代。
```

## 7. API 设计

### 7.1 任务管理

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/tasks | 创建任务 |
| GET | /api/tasks | 获取任务列表 |
| GET | /api/tasks/{id} | 获取任务详情 |
| PUT | /api/tasks/{id} | 更新任务 |
| DELETE | /api/tasks/{id} | 删除任务 |

### 7.2 执行与评估

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/tasks/{id}/executions | 执行历史 |
| GET | /api/tasks/{id}/evaluations | 评估历史 |

### 7.3 调度器

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/scheduler/start | 启动调度器 |
| POST | /api/scheduler/stop | 停止调度器 |
| GET | /api/scheduler/status | 调度器状态 |
