## 3.1 设计目标

### 3.1.1 业务需求

| 需求 | 描述 | 优先级 |
|------|------|--------|
| **S3 兼容存储** | 为 OpenClaw 提供统一的对象存储接口 | P0 |

## 3.3 高可用设计

### 3.3.1 Master 高可用

```
Master 集群（Raft 共识）:
┌──────────────┐
│  Master 1    │ ◀── Leader (处理写请求)
│  10.0.1.1    │
└──────┬───────┘
       │ Raft Heartbeat
       ▼
┌──────────────┐     ┌──────────────┐
│  Master 2    │     │  Master 3    │
│  10.0.1.2    │     │  10.0.1.3    │
│  Follower    │     │  Follower    │
└──────────────┘     └──────────────┘

故障转移流程:
1. Leader 宕机
2. Follower 检测心跳超时（~10 秒）
3. 触发选举（~5 秒）
4. 新 Leader 产生
5. 服务恢复

总恢复时间：~15 秒
```

### 3.3.2 Filer 高可用

```
Filer 集群 + MySQL 主从:
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  Filer 1     │     │  Filer 2     │     │  Filer 3     │
│  (Active)    │     │  (Active)    │     │  (Standby)   │
└──────┬───────┘     └──────┬───────┘     └──────┬───────┘

                           │ Replication
                           ▼
                  ┌─────────────────┐
                  │   MySQL Slave   │
                  │   10.0.2.2      │

```

### 3.3.3 Volume Server 冗余

```
数据复制策略（3 副本）:
Volume ID: 1001

┌─────────────────────────────────────────────────────────┐
│  副本 1: Rack 1, Volume Server 1 (10.0.3.1:9340)        │
│  副本 2: Rack 1, Volume Server 2 (10.0.3.2:9340)        │
│  副本 3: Rack 2, Volume Server 4 (10.0.4.1:9340)        │
└─────────────────────────────────────────────────────────┘

写入流程:
1. Client → Master 申请 Volume ID
2. Master 返回 3 个副本位置
3. Client 并行写入 3 个副本
4. 任意 2 个成功即返回成功

读取流程:
1. Client → Master 查询 Volume 位置
2. Master 返回最近的可读副本
3. Client 直接读取
```

## 3.4 数据存储流设计

### 3.4.1 写入流程（完整）

```
┌─────────────────────────────────────────────────────────────────────┐
│                        写入流程                                      │
└─────────────────────────────────────────────────────────────────────┘

1. Client 请求
   PUT /bucket/object
   │
   ▼
2. HAProxy 负载均衡
   选择 S3 Gateway 1
   │
   ▼
3. S3 Gateway 认证
   验证 IAM 权限
   │
   ▼
4. Filer 创建元数据
   POST /buckets/bucket/object
   │
   ▼
5. Master 分配 Volume
   返回 Volume ID + 副本位置
   │
   ▼
6. 并行写入副本
   Volume Server 1, 2, 3
   │
   ▼
7. 确认写入成功
   ≥2 副本成功即返回
   │
   ▼
8. Filer 更新状态
   标记对象为可用
   │
   ▼
9. 返回 ETag
   200 OK
```

### 3.4.2 读取流程（完整）

```
┌─────────────────────────────────────────────────────────────────────┐
│                        读取流程                                      │
└─────────────────────────────────────────────────────────────────────┘

1. Client 请求
   GET /bucket/object
   │
   ▼
2. HAProxy 负载均衡
   选择 S3 Gateway 2
   │
   ▼
3. S3 Gateway 认证
   验证 IAM 权限
   │
   ▼
4. Filer 查询元数据
   GET /buckets/bucket/object
   返回 File ID
   │
   ▼
5. Master 查询位置
   返回 Volume Server 地址
   │
   ▼
6. 读取数据
   从最近副本流式读取
   │
   ▼
7. 返回数据
   200 OK + Body
```

### 3.4.3 删除流程

```
1. Client 请求
   DELETE /bucket/object
   │
   ▼
2. S3 Gateway 认证
   │
   ▼
3. Filer 标记删除
   软删除（可恢复）
   │
   ▼
4. Volume 标记删除
   空间待回收
   │
   ▼
5. 后台 Vacuum
   定期清理已删除数据
```

### 3.5.2 备份计划

| 数据类型 | 备份方式 | 频率 | 保留期 |
|---------|---------|------|--------|
| **Master 元数据** | Raft 日志 | 实时 | 永久 |
| **Filer 元数据** | MySQL Dump | 每日 | 30 天 |
| **Volume 数据** | 异步复制 | 实时 | 与生产一致 |
| **IAM 配置** | 配置文件备份 | 每小时 | 7 天 |

### 3.6.3 告警规则

```yaml
# Prometheus AlertManager 配置
groups:
  - name: seaweedfs
    rules:
      # Master 告警
      - alert: MasterDown
        expr: up{job="seaweedfs-master"} == 0
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "SeaweedFS Master 节点宕机"
          
      # Volume 磁盘空间告警
      - alert: VolumeDiskLow
        expr: volume_disk_free_bytes / volume_disk_total_bytes < 0.2
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "Volume Server 磁盘空间不足 20%"
          
      # S3 错误率告警
      - alert: S3HighErrorRate
        expr: rate(s3_errors_total[5m]) / rate(s3_requests_total[5m]) > 0.01
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "S3 Gateway 错误率超过 1%"
          
      # Filer 连接数告警
      - alert: FilerHighConnections
        expr: filer_active_connections > 1000
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "Filer 连接数过高"
```

### 3.7.2 集成代码示例

```python
# OpenClaw 存储模块集成示例
import boto3
from typing import Optional

class SeaweedFSStorage:
    """SeaweedFS S3 存储封装"""
    
    def __init__(self):
        self.s3 = boto3.client(
            's3',
            endpoint_url='http://s3.internal:8333',
            aws_access_key_id=os.getenv('SEAWEED_ACCESS_KEY'),
            aws_secret_access_key=os.getenv('SEAWEED_SECRET_KEY'),
            region_name='us-east-1'
        )
        self.bucket = 'openclaw-storage'
    
    async def upload_workspace(self, agent_id: str, workspace_path: str):
        """上传 Agent 工作空间"""
        key = f"workspaces/{agent_id}/{os.path.basename(workspace_path)}"
        self.s3.upload_file(workspace_path, self.bucket, key)
        return key
    
    async def download_workspace(self, agent_id: str, local_path: str):
        """下载 Agent 工作空间"""
        key = f"workspaces/{agent_id}"
        self.s3.download_file(self.bucket, key, local_path)
    
    async def upload_skill(self, skill_name: str, skill_path: str):
        """上传技能包"""
        key = f"skills/{skill_name}.tar.gz"
        self.s3.upload_file(skill_path, self.bucket, key)
        return f"s3://{self.bucket}/{key}"
    
    async def store_logs(self, session_id: str, logs: bytes):
        """存储会话日志"""
        key = f"logs/{session_id}.jsonl"
        self.s3.put_object(
            Bucket=self.bucket,
            Key=key,
            Body=logs,
            ContentType='application/jsonl'
        )
```

## 3.9 小结

### 架构要点

1. **分层设计** - 接入层、网关层、元数据层、存储层清晰分离
2. **高可用** - 关键组件多副本，Raft 共识保证一致性
3. **可扩展** - Volume Server 可独立水平扩展
4. **易监控** - Prometheus + Grafana 完整监控体系
5. **安全** - 网络隔离、IAM 认证、审计日志

### 部署建议

**最小高可用部署**（开发/测试）：
- Master: 3 节点
- Filer: 2 节点 + MySQL 主从
- S3 Gateway: 2 节点
- Volume Server: 3 节点（3 副本）
- 总计：~10 节点

**生产部署**：
- Master: 3 节点（跨机架）
- Filer: 3 节点 + MySQL 集群
- S3 Gateway: 3 节点 + LB
- Volume Server: N 节点（根据容量）
- 备份集群：1 套
- 总计：~20+ 节点
