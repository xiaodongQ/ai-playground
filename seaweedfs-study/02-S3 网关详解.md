# 02-S3 网关详解 - S3 API 实现原理

> 🪣 深入剖析 SeaweedFS S3 网关的实现机制和 API 兼容性

## 2.1 S3 网关概述

### 2.1.1 架构位置

```
┌─────────────────────────────────────────────────────────────────┐
│                      S3 Client Applications                      │
│         (AWS SDK / s3cmd / mc / rclone / custom apps)           │
└────────────────────────────┬────────────────────────────────────┘
                             │ S3 API (HTTP/REST)
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│                      S3 Gateway Layer                            │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │  S3 API Handler  │  IAM Auth  │  Bucket Mapping  │  ...   │  │
│  └───────────────────────────────────────────────────────────┘  │
└────────────────────────────┬────────────────────────────────────┘
                             │ Internal API
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│                         Filer Layer                              │
│              (Directory Structure + Metadata)                    │
└────────────────────────────┬────────────────────────────────────┘
                             │ Volume API
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│                      Storage Layer                               │
│            Master Server + Volume Servers                        │
└─────────────────────────────────────────────────────────────────┘
```

### 2.1.2 核心职责

| 职责 | 说明 |
|------|------|
| **协议转换** | 将 S3 REST API 转换为 SeaweedFS 内部操作 |
| **身份认证** | 实现 AWS IAM 兼容的用户和策略管理 |
| **Bucket 映射** | 将 S3 Bucket 映射到 Filer 目录 |
| **对象存储** | 将 S3 Object 映射到 SeaweedFS File |
| **元数据管理** | 处理 Object 元数据（Content-Type、自定义标签等） |

### 2.1.3 启动配置

```bash
# 独立启动 S3 Gateway
weed s3 \
  -filer="filer-host:8888" \
  -ip="s3-host" \
  -port=8333 \
  -domainName="s3.example.com" \
  -allowedOrigins="*"

# 通过 weed server 一键启动（包含 S3）
weed server \
  -dir=/data \
  -s3 \
  -filer=true \
  -master=true \
  -volume=true
```

## 2.2 S3 API 兼容性

### 2.2.1 支持的 API 列表

#### Bucket 操作

| API | 状态 | 说明 |
|-----|------|------|
| ListBuckets | ✅ | 列出所有 Bucket |
| CreateBucket | ✅ | 创建 Bucket |
| DeleteBucket | ✅ | 删除 Bucket |
| HeadBucket | ✅ | 检查 Bucket 是否存在 |
| ListObjects | ✅ | 列出 Bucket 内对象 |
| ListObjectsV2 | ✅ | 列出对象（新版） |

#### Object 操作

| API | 状态 | 说明 |
|-----|------|------|
| PutObject | ✅ | 上传对象 |
| GetObject | ✅ | 下载对象 |
| DeleteObject | ✅ | 删除对象 |
| DeleteObjects | ✅ | 批量删除 |
| HeadObject | ✅ | 获取对象元数据 |
| CopyObject | ✅ | 复制对象 |
| PutObjectCopy | ✅ | 复制对象（兼容） |
| GetObjectTagging | ✅ | 获取对象标签 |
| PutObjectTagging | ✅ | 设置对象标签 |

#### Multipart Upload

| API | 状态 | 说明 |
|-----|------|------|
| CreateMultipartUpload | ✅ | 创建分片上传 |
| UploadPart | ✅ | 上传分片 |
| ListParts | ✅ | 列出已上传分片 |
| CompleteMultipartUpload | ✅ | 完成分片上传 |
| AbortMultipartUpload | ✅ | 取消分片上传 |

#### 预签名 URL

| API | 状态 | 说明 |
|-----|------|------|
| Presigned URL | ✅ | 生成临时访问 URL |
| Presigned POST | ✅ | 生成表单上传策略 |

### 2.2.2 已知限制

| 限制 | 影响 |  workaround |
|------|------|-------------|
| **部分 ACL API** | 部分 ACL 操作不完全兼容 | 使用 IAM 策略替代 |
| **对象锁定** | 不支持 WORM（Write Once Read Many） | 使用 Bucket 策略限制删除 |
| **智能分层** | 不支持自动存储分层 | 手动配置云同步 |
| **分析功能** | 不支持 Storage Lens 等 | 使用监控 API |
| **事件通知** | 有限支持 | 使用 Webhook 或轮询 |

## 2.3 数据映射机制

### 2.3.1 Bucket 到目录的映射

```
S3 概念          →    SeaweedFS 概念
─────────────────────────────────────
Bucket               /buckets/{bucket_name}/
Object               /buckets/{bucket_name}/{key}
Object Version       /buckets/{bucket_name}/{key}#{version}
Multipart Upload     /buckets/{bucket_name}/.uploads/{uploadId}/
```

### 2.3.2 对象存储流程

#### 上传对象 (PutObject)

```
1. Client 发送 PUT /bucket/key
         │
2. S3 Gateway 验证权限
         │
3. 写入 Filer 临时文件
         │
4. Filer 分配 File ID
         │
5. 写入 Volume Server
         │
6. 更新 Filer 元数据
         │
7. 返回 ETag
```

#### 下载对象 (GetObject)

```
1. Client 发送 GET /bucket/key
         │
2. S3 Gateway 验证权限
         │
3. 查询 Filer 获取 File ID
         │
4. 查询 Master 获取 Volume 位置
         │
5. 从 Volume Server 读取数据
         │
6. 流式返回给 Client
```

### 2.3.3 大文件处理

对于大文件（>5GB），SeaweedFS 使用分片上传：

```
大文件上传流程:
┌──────────────┐
│ 1. 创建上传   │ CreateMultipartUpload → 返回 uploadId
└──────┬───────┘
       │
┌──────▼───────┐
│ 2. 上传分片   │ UploadPart(partNumber, data) → 返回 ETag
│    (并行)    │ 可并发上传多个分片
└──────┬───────┘
       │
┌──────▼───────┐
│ 3. 完成上传   │ CompleteMultipartUpload([ETag...])
└──────┬───────┘
       │
┌──────▼───────┐
│ 4. 合并文件   │ Filer 合并所有分片为完整对象
└──────────────┘
```

## 2.4 IAM 身份认证

### 2.4.1 认证架构

```
┌─────────────────────────────────────────────────────────┐
│                    IAM System                            │
├─────────────────────────────────────────────────────────┤
│  ┌──────────┐  ┌──────────┐  ┌──────────┐             │
│  │  Users   │  │  Groups  │  │ Policies │             │
│  │  (用户)  │  │  (组)    │  │  (策略)  │             │
│  └──────────┘  └──────────┘  └──────────┘             │
│       │              │              │                   │
│       └──────────────┼──────────────┘                   │
│                      │                                  │
│              ┌───────▼───────┐                          │
│              │  Access Keys  │                          │
│              │  (AK/SK)      │                          │
│              └───────────────┘                          │
└─────────────────────────────────────────────────────────┘
```

## 2.5 客户端配置示例

### 2.5.1 AWS CLI

```bash
# 配置凭证
aws configure set aws_access_key_id admin
aws configure set aws_secret_access_key key
aws configure set default.region us-east-1

# 配置 S3 端点
aws configure set default.s3.endpoint_url http://localhost:8333

# 测试命令
aws s3 ls --endpoint-url http://localhost:8333
aws s3 mb s3://my-bucket --endpoint-url http://localhost:8333
aws s3 cp file.txt s3://my-bucket/ --endpoint-url http://localhost:8333
```

### 2.5.2 s3cmd

```ini
# ~/.s3cfg
[default]
access_key = admin
secret_key = key
host_base = localhost:8333
host_bucket = localhost:8333
use_https = False
signature_v2 = False
```

```bash
s3cmd ls s3://
s3cmd mb s3://my-bucket
s3cmd put file.txt s3://my-bucket/
```

### 2.5.3 mc (MinIO Client)

```bash
# 添加别名
mc alias set seaweed http://localhost:8333 admin key

# 操作命令
mc ls seaweed/
mc mb seaweed/my-bucket
mc cp file.txt seaweed/my-bucket/
mc mirror ./local-dir seaweed/my-bucket/
```

### 2.5.4 Python SDK (boto3)

```python
import boto3

# 配置客户端
s3 = boto3.client(
    's3',
    endpoint_url='http://localhost:8333',
    aws_access_key_id='admin',
    aws_secret_access_key='key',
    region_name='us-east-1'
)

# 创建 Bucket
s3.create_bucket(Bucket='my-bucket')

# 上传对象
s3.upload_file('local-file.txt', 'my-bucket', 'remote-file.txt')

# 下载对象
s3.download_file('my-bucket', 'remote-file.txt', 'downloaded.txt')

# 列出对象
response = s3.list_objects_v2(Bucket='my-bucket')
for obj in response.get('Contents', []):
    print(obj['Key'])
```

### 2.5.5 Go SDK

```go
package main

import (
    "github.com/aws/aws-sdk-go/aws"
    "github.com/aws/aws-sdk-go/aws/credentials"
    "github.com/aws/aws-sdk-go/aws/session"
    "github.com/aws/aws-sdk-go/service/s3"
)

func main() {
    sess := session.Must(session.NewSession(&aws.Config{
        Endpoint:    aws.String("http://localhost:8333"),
        Region:      aws.String("us-east-1"),
        Credentials: credentials.NewStaticCredentials("admin", "key", ""),
    }))

    svc := s3.New(sess)

    // 创建 Bucket
    svc.CreateBucket(&s3.CreateBucketInput{
        Bucket: aws.String("my-bucket"),
    })

    // 上传对象
    svc.PutObject(&s3.PutObjectInput{
        Bucket: aws.String("my-bucket"),
        Key:    aws.String("my-key"),
        Body:   strings.NewReader("Hello, SeaweedFS!"),
    })
}
```

## 2.7 性能优化

### 2.7.1 连接池配置

```python
# boto3 连接池优化
from botocore.config import Config

config = Config(
    max_pool_connections=50,
    connect_timeout=5,
    read_timeout=30,
    retries={'max_attempts': 3}
)

s3 = boto3.client('s3', config=config, ...)
```

### 2.7.2 分片上传优化

```python
# 大文件分片上传
from boto3.s3.transfer import TransferConfig

config = TransferConfig(
    multipart_threshold=8 * 1024 * 1024,  # 8MB
    multipart_chunksize=8 * 1024 * 1024,  # 8MB
    max_concurrency=10,
    use_threads=True
)

s3.upload_file('large-file.bin', 'bucket', 'key', Config=config)
```

### 2.8.2 指标暴露

SeaweedFS 暴露 Prometheus 格式的指标：

```bash
# 访问指标端点
curl http://localhost:8333/metrics
```

关键指标：
- `s3_requests_total` - S3 请求总数
- `s3_request_duration_seconds` - 请求延迟
- `s3_uploaded_bytes` - 上传字节数
- `s3_downloaded_bytes` - 下载字节数

### 2.8.3 常见问题排查

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| **403 Forbidden** | 凭证错误或权限不足 | 检查 AK/SK 和 IAM 策略 |
| **404 Not Found** | Bucket 或对象不存在 | 检查路径和 Bucket 名称 |
| **500 Internal Error** | 后端服务异常 | 检查 Filer 和 Volume Server 状态 |
| **慢上传** | 网络或磁盘瓶颈 | 检查网络带宽和磁盘 IO |
| **分片上传失败** | 分片顺序或 ETag 错误 | 检查 UploadPart 返回的 ETag |

## 2.9 小结

### 核心要点

1. **高度兼容** - 支持主流 S3 API，可无缝替换 AWS S3
2. **灵活认证** - 支持环境变量、配置文件、Admin UI 多种配置方式
3. **映射清晰** - Bucket→目录，Object→文件，概念对应清晰
4. **分片上传** - 支持大文件分片上传，可并发加速
5. **易于集成** - 所有主流 S3 SDK 均可直接使用

### 最佳实践

✅ **推荐**：
- 生产环境使用独立 IAM 用户，避免使用 admin
- 大文件使用分片上传，设置合适的并发数
- 配置 CORS 和生命周期策略优化性能
- 定期备份 IAM 配置

❌ **避免**：
- 在生产环境使用默认凭证
- 单线程上传大文件
- 忽略错误重试逻辑
- 不监控 S3 网关性能
