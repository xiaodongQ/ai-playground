# 01-项目概览

> 理解 nanoClaw 的定位、目标和核心特性

## 1.1 项目定位

### 什么是 nanoClaw？

nanoClaw 是一个**超轻量级、默认安全、2 分钟安装**的 AI 个人助手框架，通过 Telegram 提供 24/7 不间断服务。

```
OpenClaw:  430,000 行代码，复杂配置，安全隐患
nanoClaw:   ~3,000 行代码，默认安全，2 分钟安装
```

### 设计灵感

nanoClaw 受 OpenClaw 启发，但针对其三大痛点进行了重新设计：

| 痛点 | OpenClaw | nanoClaw 解决方案 |
|------|----------|------------------|
| **过于复杂** | 43 万行代码，难以理解和维护 | ~3000 行，每个文件都有行数预算 |
| **安装困难** | 需要复杂的环境配置 | 2 分钟安装，交互式向导 |
| **安全隐患** | 文件系统访问无限制 | 6 层安全防护，默认沙箱 |

## 1.2 核心目标

nanoClaw 的设计遵循四大支柱（4F 原则）：

### 1. SECURE（安全）

> 用户将 shell 访问权限信任给我们，每个输入都是敌对的，直到被证明安全。

**防御深度策略**：如果一层失效，下一层会捕获。绝不在安全问题上走捷径。

**6 层安全防护**：
1. **FileGuard** - 文件访问限制在工作区
2. **ShellSandbox** - 阻止危险命令，确认破坏性操作
3. **PromptGuard** - 检测和清理提示词注入
4. **SessionBudget** - 速率限制和成本控制
5. **AuditLog** - 记录所有代理操作
6. **SecurityDoctor** - 验证安装安全性

### 2. LIGHTWEIGHT（轻量）

> 目标：总计约 3000 行代码。没有 2 个以上具体用途的抽象，没有仅委托的包装类。

**代码规模控制**：
- 核心模块：~500 行
- 工具系统：~400 行
- 安全模块：~600 行
- Dashboard: ~300 行
- 其他：~1200 行

**依赖最小化**：
```python
# 核心依赖仅 6 个
dependencies = [
    "aiohttp>=3.9",           # 异步 HTTP
    "python-telegram-bot>=20.0",  # Telegram 集成
    "click>=8.0",             # CLI
    "pydantic>=2.0",          # 数据验证
    "html2text>=2024.2",      # HTML 转文本
    "croniter>=1.3",          # 定时任务
]
```

### 3. FAST（快速）

> 用户在 Telegram 等待。目标：简单查询首响应 <2 秒。

**性能优化策略**：
- **共享连接池**: 一个 aiohttp session 复用
- **异步一切**: 所有 I/O 都是异步
- **懒加载**: chromadb、croniter 在函数内导入
- **智能缓存**: 会话级缓存避免重复请求

**响应时间目标**：

| 场景 | 目标时间 | 实现方式 |
|------|---------|---------|
| 简单问题，无工具 | <2s | 1 次 LLM 调用，紧凑上下文 |
| 单次工具调用 | <4s | LLM + 工具 + LLM |
| 多工具并行 (3 个) | <5s | LLM + 并行工具 + LLM |
| 研究任务 (5+ 迭代) | <15s | 并行 + 缓存重复 |

### 4. EFFECTIVE（有效）

> 代理必须真正解决问题。偏向行动：调用工具，不要描述你可以调用什么工具。

**迭代纪律**：
```
1-2 次迭代：简单问答，单次工具 - 理想
3-4 次迭代：多工具研究 - 正常
5+ 次迭代：记录警告，代理可能挣扎
10+ 次迭代：强制停止，返回部分结果
```

## 1.3 与 OpenClaw 的关系和区别

### 关系

nanoClaw 是 OpenClaw 的**精神继承者**，不是分支或 fork。它借鉴了 OpenClaw 的核心理念，但从零开始重新设计，解决了 OpenClaw 的根本性问题。

### 核心区别

| 维度 | OpenClaw | nanoClaw |
|------|----------|----------|
| **代码规模** | 430,000 行 | ~3,000 行 |
| **技能数量** | 100+ | 5 个内置 + 自定义 |
| **工具注入** | 全部发送 (~5000 tokens) | 动态选择 (~800 tokens) |
| **历史记录** | 50 条完整发送 | 4-15 条智能窗口 |
| **工具输出** | 无截断 | 按工具类型截断 |
| **内存提取** | 每条消息 | 仅 30-40% 消息 |
| **日均成本** | $3-5 | $0.50-1.00 |
| **启动时间** | 较慢 | <2 秒 |

### Token 效率对比

| 组件 | OpenClaw | nanoClaw | 节省 |
|------|----------|----------|------|
| 系统提示 | ~800 tokens | ~300 tokens | 62% |
| 工具模式 | ~5000 tokens | ~800 tokens | 84% |
| 历史记录 | ~15000 tokens | ~3000 tokens | 80% |
| 工具输出 | ~8000 tokens | ~2000 tokens | 75% |
| 内存提取 | 每条消息 | 30-40% 消息 | 60-70% |
| **3 次对话总成本** | **~$0.05-0.10** | **~$0.01-0.02** | **75-80%** |

## 1.4 核心特性详解

### 1. 默认安全

**文件系统沙箱**：
```python
# FileGuard 限制所有文件操作在工作区
class FileGuard:
    BLOCKED_COMPONENTS = [".env", ".git", ".ssh", ".gnupg", ".aws"]
    BLOCKED_FILENAMES = ["id_rsa", "config.json"]
    
    def validate_path(self, path: str) -> Path:
        resolved = (self.workspace / path).resolve()
        # 检查是否在工作区内
        resolved.relative_to(self.workspace)  # 越界抛出 SecurityError
        return resolved
```

**Shell 命令过滤**：
```python
# 三层过滤：BLOCKED → CONFIRM → EXECUTE
BLOCKED_PATTERNS = [
    r"rm\s+(-[a-zA-Z]*)?rf\s+[/~]",  # rm -rf /
    r"mkfs\.",  # 格式化磁盘
    r"curl.*\|\s*(ba)?sh",  # 管道执行
]

CONFIRM_PATTERNS = [
    r"\brm\s",  # 任何删除
    r"pip\s+install",  # 安装包
    r"sudo\s",  # 提权
]
```

**提示词注入防御**：
```python
INJECTION_PATTERNS = [
    r"ignore\s+(previous|above|all)\s+(instructions|rules)",
    r"you\s+are\s+now\s+",
    r"disregard\s+previous",
    r"system\s*:\s*",
]

def sanitize_tool_output(self, tool_name: str, raw_output: str) -> str:
    detected, matched = self.check_injection(raw_output)
    if detected:
        return f"[WARNING: 提示注入检测]\n{raw_output}"
    return raw_output
```

### 2. 模型无关

支持所有主流 LLM 提供商：

| 提供商 | 模型 | 获取 API Key |
|--------|------|-------------|
| OpenRouter | 所有主流模型 | [openrouter.ai/keys](https://openrouter.ai/keys) |
| DeepSeek | deepseek-chat, deepseek-reasoner | [platform.deepseek.com](https://platform.deepseek.com) |
| Anthropic | Claude Sonnet/Opus/Haiku | [console.anthropic.com](https://console.anthropic.com) |
| OpenAI | GPT-5 系列 | [platform.openai.com](https://platform.openai.com) |
| 本地 | Ollama, LM Studio | - |

**统一客户端**：
```python
class LLMClient:
    BASE_URLS = {

        "openai": "https://api.openai.com/v1",
    }
    
    async def chat(self, messages, tools=None, model=None):
        # 自动适配不同提供商的 API 格式
        if self.provider == "anthropic":
            return self._parse_anthropic_response(data)
        return self._parse_openai_response(data)
```

### 3. 超轻量级

**代码组织**：
```
nanoclaw/
├── core/           # 核心：agent、llm、context、config
├── tools/          # 工具：web、shell、files、memory、spawn
├── security/       # 安全：sandbox、prompt_guard、budget、audit
├── memory/         # 记忆：SQLite 存储
├── channels/       # 渠道：telegram、console、gateway
├── cron/           # 定时：调度器
└── dashboard/      # 仪表板：本地 Web UI
```

**最小依赖**：
- 使用 stdlib：`sqlite3`、`json`、`asyncio`、`pathlib`、`re`
- 无重型框架：没有 langchain、llamaindex、fastapi

### 6. 持久记忆

**SQLite 存储**：
```python
# 自动保存重要事实
async def _extract_memories(self, user_message: str, response: str):
    if len(user_message) < 20:
        return  # 跳过短消息
    
    # 检测个人事实
    triggers = ["my name", "i work", "i live", "i prefer"]
    if any(t in user_message.lower() for t in triggers):
        facts = await self.llm.extract_facts(user_message, response)
        for fact in facts[:3]:
            await self.memory.save_memory(fact, category="auto")
```

**智能检索**：
```python
# FTS5 全文搜索
def _fts_query(self, query: str) -> str:
    words = query.split()
    return " OR ".join(w for w in words if len(w) > 2)

# 仅返回最相关的 5 条记忆
memories = await memory.search(query, limit=5)
```

## 1.6 小结

### 核心要点

1. **定位清晰**: nanoClaw 是 OpenClaw 的轻量级替代，专注个人助手场景
2. **4F 原则**: SECURE、LIGHTWEIGHT、FAST、EFFECTIVE 指导所有设计决策
3. **安全优先**: 6 层防护，默认沙箱，无开放端口
4. **Token 效率**: 通过动态工具注入、智能窗口、输出压缩节省 75-80% Token
5. **极简主义**: ~3000 行代码，6 个核心依赖，2 分钟安装

### 关键数据

- 代码行数：~3,000 vs OpenClaw 的 430,000
- Token 成本：$0.50-1.00/天 vs $3-5/天
- 启动时间：<2 秒
- 响应时间：简单查询 <2 秒
