```
              ┌────────────────┼────────────────┐
              │                │                │

      ┌───────────────┐ ┌───────────────┐ ┌───────────────┐
      │   Master 2    │ │   Master 3    │ │   Master 4    │
      │   (Follower)  │ │   (Follower)  │ │   (Follower)  │

故障转移:
1. Leader 宕机
2. Follower 检测超时 (~10s)
3. 触发选举 (~5s)
4. 新 Leader 产生
5. 服务恢复

总 RTO: ~15 秒
```

## A.5 存储架构

### A.5.1 Volume 文件结构

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    Volume 文件结构                                        │
└─────────────────────────────────────────────────────────────────────────┘

Volume ID: 1001

/data/volume1/
├── 1001.dat          # 数据文件（追加写入）
│   ┌─────────────────────────────────────────────────────────┐
│   │  ┌──────────┐  ┌──────────┐  ┌──────────┐            │
│   │  │  File 1  │  │  File 2  │  │  File 3  │   ...      │
│   │  │ Header   │  │ Header   │  │ Header   │            │
│   │  │ + Data   │  │ + Data   │  │ + Data   │            │
│   │  └──────────┘  └──────────┘  └──────────┘            │
│   └─────────────────────────────────────────────────────────┘
│
├── 1001.idx          # 索引文件（内存加载）
│   ┌─────────────────────────────────────────────────────────┐
│   │  FileID → Offset + Size + Cookie + CRC                 │
│   │  FileID → Offset + Size + Cookie + CRC                 │
│   │  ...                                                    │
│   └─────────────────────────────────────────────────────────┘
│
├── 1001.idx.v2       # 索引版本（可选）
└── 1001.meta         # 元数据（可选）

文件定位:
1. 通过 FileID 在 .idx 中查找
2. 获取 Offset 和 Size
3. 在 .dat 中 seek(offset)
4. read(size) 获取数据

O(1) 时间复杂度！
```

```

Volume ID  │  Replica 1      │  Replica 2      │  Replica 3

1001       │  10.0.4.1:9340  │  10.0.4.2:9340  │  10.0.5.1:9340

1. 同一 Volume 的副本分布在不同服务器

4. 写入时自动选择最优副本
```

```yaml
        for: 1m
        severity: critical
        
      - alert: VolumeDiskLow
```
