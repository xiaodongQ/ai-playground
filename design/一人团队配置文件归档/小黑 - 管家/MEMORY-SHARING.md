# MEMORY-SHARING.md - 共享记忆机制

## 模式：部分共享（小黑 - 管家为中心）

小黑 - 管家存储全局记忆，子 Agent 按需读取。

---

## 一、记忆分类

### 1. 全局记忆（小黑 - 管家存储）

**位置**: `/root/.openclaw/workspace/MEMORY.md`

**内容**：
- 用户基本信息（姓名、称呼、偏好）
- 项目背景与上下文
- 长期目标和计划
- 重要决策记录
- 跨 Agent 协作历史

**访问权限**：所有 Agent 可读，仅小黑可写

### 2. 专业记忆（各 Agent 独立存储）

**位置**: `/root/.openclaw/workspace-{agent}/memory/`

**内容**：
- 专业领域对话历史
- 领域特定偏好（如代码风格、写作风格）
- 专业项目进度

**访问权限**：仅对应 Agent 可读写

### 3. 会话记忆（按会话隔离）

**位置**: `/root/.openclaw/agents/{agent}/sessions/`

**内容**：
- 当前会话的完整对话历史
- 临时上下文

**访问权限**：仅对应 Agent 可访问

---

## 二、记忆共享流程

### 小黑 → 子 Agent（传递上下文）

```
用户请求 → 小黑分析 → 需要路由 → 
小黑提取关键上下文 → 传递给目标 Agent → 
目标 Agent 处理 → 返回结果
```

**传递内容**：
- 用户当前需求摘要
- 相关背景信息
- 之前的协作历史（如有）

### 子 Agent → 小黑（更新全局记忆）

```
子 Agent 完成任务 → 发现重要信息 → 
通知小黑更新 MEMORY.md → 小黑确认并写入
```

**更新内容**：
- 新学到的用户偏好
- 项目进展里程碑
- 重要决策

## 四、记忆访问规范

### 小黑 - 管家

```
✅ 可读取：所有记忆文件
✅ 可写入：MEMORY.md、自有记忆
✅ 可删除：过期会话记忆
```

### 子 Agent

```
✅ 可读取：MEMORY.md（全局记忆）、自有记忆
❌ 不可写入：MEMORY.md（需通过小黑更新）
✅ 可写入：自有记忆目录
✅ 可请求：通过小黑获取其他 Agent 记忆摘要
```

---

## 五、隐私保护

### 敏感信息隔离

| 信息类型 | 存储位置 | 访问权限 |
|---------|---------|---------|
| 财务数据 | Fin 独立记忆 | 仅 Fin + 小黑 |
| 代码/项目 | Dev 独立记忆 | Dev + 小黑 |
| 学习笔记 | Edu 独立记忆 | Edu + 小黑 |
| 文章内容 | Wri 独立记忆 | Wri + 小黑 |
| 用户基本信息 | MEMORY.md | 全部 Agent |

### 保密模式

用户可要求开启保密模式：
- 对话不记录到 MEMORY.md
- 仅保存在当前会话记忆
- 会话结束后可选择删除

---

## 六、记忆维护

### 定期清理（小黑负责）

- **每日**：检查会话记忆是否已归档
- **每周**：整理 MEMORY.md，移除过期信息
- **每月**：生成记忆摘要，归档旧文件

### 记忆压缩

当记忆文件过大时：
1. 提取关键信息到 MEMORY.md
2. 归档原始文件到 `memory/archive/`
3. 更新索引

---

## 七、实现建议

### 方案 A：文件共享（推荐）

所有 Agent 通过文件系统共享 MEMORY.md：
- 优点：简单直接，无需额外服务
- 缺点：需要处理并发写入

### 方案 B：会话传递

小黑通过 `sessions_send` 传递上下文：
- 优点：符合 OpenClaw 架构
- 缺点：增加通信开销

### 方案 C：数据库存储

使用轻量数据库（如 SQLite）：
- 优点：支持复杂查询，并发安全
- 缺点：增加依赖

**当前选择：方案 A**（文件共享）
