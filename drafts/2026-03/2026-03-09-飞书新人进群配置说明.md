### 场景 2：新人想私聊 Agent

**⚠️ 需要配置**

如果新人想单独私聊某个 Agent，需要将他的用户 ID 添加到对应 Agent 的 `allowFrom` 列表中。

#### 操作步骤

1. **获取新人的飞书用户 ID**（格式：`ou_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`）
   
   方法：
   - 新人进群后发一条消息，从消息元数据中获取
   - 在飞书后台查看
   - 让新人私聊我一次，自动记录

2. **更新配置文件**

   编辑 `/root/.openclaw/openclaw.json`：
   
   ```json
   {
     "channels": {
       "feishu": {
         "accounts": {
           "main": {
             "allowFrom": [
               "ou_9197e1d0c38ef2aeec4eeb9866c7b7f7",  // 原有用户
               "ou_新人的用户 ID"                        // 新增用户
             ]
           },
           "dev-agent": {
             "allowFrom": [
               "ou_9197e1d0c38ef2aeec4eeb9866c7b7f7",
               "ou_新人的用户 ID"
             ]
           }
           // ... 其他 Agent 同理
         }
       }
     }
   }
   ```

3. **重启 Gateway**

   ```bash
   openclaw gateway restart
   ```

## 常见问题

### Q1: 新人进群后 @Agent 没反应？

**可能原因**：
1. 群聊 ID 不在 `groupAllowFrom` 列表中
2. Gateway 未运行
3. 飞书 App Secret 过期

**排查步骤**：
```bash
# 1. 检查 Gateway 状态
openclaw gateway status

# 2. 检查群聊配置
cat /root/.openclaw/openclaw.json | grep -A 5 'groupAllowFrom'

# 3. 查看日志
tail -f /root/.openclaw/logs/*.log
```

---

### Q2: 如何获取用户的飞书 ID？

**方法 1**: 从消息元数据获取（推荐）
- 用户发消息后，查看消息元数据中的 `sender_id` 字段

**方法 2**: 飞书后台
- 登录飞书开放平台 → 应用管理 → 用户管理

**方法 3**: 让私聊触发
- 用户私聊机器人一次，自动记录到凭证文件

---

### Q3: 多人协作需要为每个人配置吗？

**建议**：
- 如果只在群聊使用 → 不需要，群聊 ID 在白名单即可
- 如果需要私聊 → 需要为每个人配置 `allowFrom`

## 安全建议

### 允许列表管理

- ✅ 定期审查 `allowFrom` 列表
- ✅ 离职/退出人员及时移除
- ✅ 敏感操作（如代码执行、财务查询）设置二次确认

### 群聊管理

- ✅ 设置群聊为私有（不公开邀请）
- ✅ 开启群聊审核（新人入群需批准）
- ✅ 记录群聊成员变更日志
