# AI Task System V3 设计文档（CodeBuddy 个人内网专属版）

**文档版本**：V1.0  
**更新日期**：2026-04-17  
**参考来源**：豆包 AI 生成版  
**适用环境**：纯内网单机无容器环境、Python 3.10+、CodeBuddy CLI 已就绪

---

## 一、项目定位

专为个人开发者打造的、深度适配腾讯 CodeBuddy CLI 的 AI 任务自动化执行与管理平台。

核心目标：
1. **原生适配**：100% 遵循腾讯 CodeBuddy 官方无头模式规范
2. **极简部署**：单机无容器、零配置一键启动，纯内网离线可用
3. **全链路可观测**：任务执行状态、日志全量实时同步
4. **双友好沉淀**：固定目录结构、标准化 Markdown 格式，人和 AI 都能零歧义读取

---

## 二、核心设计原则

| 原则 | 要求 |
|------|------|
| 本地优先 | 所有数据 100% 存储在本地，无强制云服务依赖 |
| 极简可靠 | 最小依赖、最少服务组件，无容器、单机 Python 即可运行 |
| 规范兼容 | 命令风格、执行参数 100% 兼容 CodeBuddy 官方 CLI 规范 |
| 全链路可追溯 | 任务全生命周期状态、日志、产出、版本永久沉淀 |
| 双友好设计 | 所有目录、文件、接口均采用固定结构化格式 |

## 四、核心模块

### 4.1 调度器 (scheduler.py)
- **双模式**：新建即执行（默认）+ 定时轮询（可选，30 秒间隔）
- **并发控制**：最大同时执行 2 个任务
- **生命周期**：监听任务状态，完成/失败/超时后触发后续流程

### 4.2 执行器 (executor.py)
- **CLI 规范**：严格遵循 CodeBuddy 官方无头模式
- **双超时防护**：
  - 绝对超时：默认 1 小时
  - 无输出超时：默认 10 分钟
- **多轮对话**：支持 `--resume` 续跑会话

### 4.3 存储 (storage.py)
- **SQLite**：全量数据持久化
- **参数化查询**：杜绝 SQL 注入
- **自动备份**：每日凌晨备份数据库

### 4.4 实时同步 (realtime.py)
- **WebSocket**：FastAPI 原生实现，无额外依赖
- **事件全覆盖**：task_created/status_changed/log_appended/completed/failed
- **日志兜底**：实时推送 + 本地文件双写

## 六、Markdown 固定模板

### TASK.md
```markdown

## 基础信息
- 任务ID：{task_id}
- 任务标题：{title}

- 最大迭代次数：{max_iterations}

- 权限模式：{permission_mode}

{description}

- 执行结束时间：{completed_at}

{content}

## 产出文件清单
{file_list}

## Git版本记录
{git_log}
```

## 七、CLI 命令规范

```bash
# 创建并执行任务
task-cli -p "任务描述"

# 续跑任务
task-cli --resume {task_id} "优化内容"

# 任务管理
task-cli list
task-cli get {task_id}
task-cli stop {task_id}
task-cli delete {task_id}

# 调度器控制
task-cli scheduler start
task-cli scheduler stop
task-cli scheduler status
```

## 八、环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| TASK_DB_PATH | ./data/tasks.db | 数据库路径 |
| WORKSPACE_ROOT | ./workspace | 任务沙箱根目录 |
| HOST | 127.0.0.1 | 服务监听地址 |
| PORT | 8000 | 服务端口 |
| SCHEDULER_INTERVAL | 30 | 轮询间隔（秒） |
| MAX_CONCURRENT_TASKS | 2 | 最大并发任务数 |
| DEFAULT_ABSOLUTE_TIMEOUT | 3600 | 绝对超时（秒） |
| DEFAULT_NO_OUTPUT_TIMEOUT | 600 | 无输出超时（秒） |

## 九、部署步骤

1. 安装依赖：`pip install -r requirements.txt`
2. 配置环境：`cp .env.example .env`
3. 启动服务：`python main.py`
4. 访问 Web UI：http://localhost:8000
