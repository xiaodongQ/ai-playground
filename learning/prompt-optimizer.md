> 让 AI 帮你写出更好的提示词 | 自举式学习系统

## 🚀 快速开始

### 核心提示词（复制即用）

```markdown
# 提示词优化专家

你是一个专业的提示词工程师，擅长分析和改进提示词。

## 任务
帮我优化以下提示词：

**原始提示词**:
[在此粘贴]

**使用场景**: [描述]
**期望效果**: [描述]
**当前问题**: [描述]

## 输出要求
1. **问题分析** - 指出原始提示词的 3-5 个主要问题
2. **优化版本** - 提供 3 个版本：
   - 🟢 保守版（最小改动）
   - 🟡 平衡版（推荐）
   - 🔴 激进版（最大优化）
3. **改进说明** - 解释每个版本的改进点
4. **使用建议** - 何时使用哪个版本

## 评估标准
- 清晰度：目标是否明确
- 完整性：信息是否充分
- 可执行性：AI 能否理解并执行
- 效率：Token 使用是否合理
```

## 📚 提示词工程原理

### 1. 提示词结构公式

```
优质提示词 = 角色 + 任务 + 约束 + 上下文 + 示例 + 输出格式
```

#### 各要素详解

| 要素 | 作用 | 示例 |
|------|------|------|
| **角色** | 设定 AI 身份 | "你是一位资深 Python 工程师" |
| **任务** | 明确要做什么 | "请审查以下代码" |
| **约束** | 限定范围 | "只关注安全性，忽略风格" |
| **上下文** | 提供背景 | "这是生产环境的支付模块" |
| **示例** | 展示期望 | "像这样输出：{格式}" |
| **输出格式** | 规范结果 | "用 Markdown 表格呈现" |

### 2. 常见问题诊断

| 问题 | 症状 | 解决方案 |
|------|------|---------|
| **模糊** | AI 输出不符合预期 | 增加具体约束和示例 |
| **冗长** | Token 消耗大，重点不突出 | 精简描述，用列表代替段落 |
| **缺失上下文** | AI 理解偏差 | 补充背景信息和边界条件 |
| **多任务混杂** | AI 顾此失彼 | 拆分成多个提示词或子任务 |
| **缺少输出规范** | 格式混乱 | 明确指定输出格式 |

## 🛠️ 优化工具箱

### 工具 1: CRISP 框架

```
C - Context (上下文)
R - Role (角色)
I - Instruction (指令)
S - Specification (规范)
P - Presentation (呈现)
```

**应用示例**:

```markdown
# 原始
"帮我写个脚本"

# CRISP 优化后
[C] 我需要自动化部署前端项目
[R] 你是一位 DevOps 工程师
[I] 编写一个 Bash 部署脚本
[S] 要求：支持回滚、日志记录、环境变量配置
[P] 输出：完整脚本 + 使用说明 + 注意事项
```

### 工具 2: 5W1H 检查法

```
What  - 要做什么？
Why   - 为什么做？
Who   - 谁来做/给谁用？
When  - 何时执行？
Where - 在哪里执行？
How   - 如何做？
```

**检查清单**:
```markdown
- [ ] What: 任务目标清晰吗？
- [ ] Why: 背景目的说明了吗？
- [ ] Who: 目标用户/执行者明确吗？
- [ ] When: 时间/时机有说明吗？
- [ ] Where: 环境/位置指定了吗？
- [ ] How: 方法/格式定义了吗？
```

### 工具 3: 迭代优化循环

```
┌──────────────┐
│  初始版本    │
└──────┬───────┘
       │
       ↓
┌──────────────┐     ┌──────────────┐
│  AI 执行      │────→│  结果评估    │
└──────┬───────┘     └──────┬───────┘
       │                    │
       │              ┌─────┴─────┐
       │              │ 符合期望？ │
       │              └─────┬─────┘
       │                    │
       │             是     │     否
       │                    │
       │                    ↓
       │            ┌──────────────┐
       │            │  问题诊断    │
       │            └──────┬───────┘
       │                   │
       └───────────────────┘
                   ↓
           ┌──────────────┐
           │  提示词改进  │
           └──────┬───────┘
                  │
                  ↓
           回到 AI 执行
```

## 🎓 高级技巧

### 技巧 1: 思维链 (Chain of Thought)

**原理**: 让 AI 展示推理过程，提高准确性

**模板**:
```markdown
请逐步思考这个问题：

1. 首先，分析...
2. 然后，考虑...
3. 接着，评估...
4. 最后，得出...

请展示你的完整推理过程，然后给出答案。
```

### 技巧 2: 少样本学习 (Few-Shot Learning)

**原理**: 提供示例，让 AI 学习模式

**模板**:
```markdown
任务：[任务描述]

示例 1:
输入：[示例输入 1]
输出：[示例输出 1]

示例 2:
输入：[示例输入 2]
输出：[示例输出 2]

示例 3:
输入：[示例输入 3]
输出：[示例输出 3]

现在请处理：
输入：[实际输入]
输出：
```

### 技巧 3: 角色嵌套

**原理**: 多层角色设定，获得更专业输出

**模板**:
```markdown
你是一位 [主角色]，正在指导一位 [子角色] 完成任务。

主角色视角：[专业视角和要求]
子角色视角：[执行层面的考虑]

请综合两个视角，给出既专业又可执行的建议。
```

### 技巧 4: 约束递增

**原理**: 逐步增加约束，精炼输出

**模板**:
```markdown
第一轮：请列出所有可能的方案
第二轮：请筛选出最可行的 3 个方案
第三轮：请详细分析最优方案
```

### 技巧 5: 反向验证

**原理**: 让 AI 自我验证输出质量

**模板**:
```markdown
请完成 [任务]，然后：
1. 检查你的输出是否满足所有要求
2. 指出可能的不足
3. 提供改进建议
4. 如果有必要，重新生成改进版本
```

## 📊 质量评估量表

### 提示词质量评分 (1-10 分)

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| **清晰度** | 25% | 10=目标极其明确，1=完全模糊 |
| **完整性** | 25% | 10=信息充分，1=严重缺失 |
| **可执行性** | 25% | 10=AI 轻松理解，1=无法执行 |
| **效率** | 15% | 10=Token 利用最优，1=大量冗余 |
| **安全性** | 10% | 10=无风险，1=高危操作 |

### 输出质量评分 (1-10 分)

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| **准确性** | 30% | 10=完全正确，1=完全错误 |
| **完整性** | 25% | 10=覆盖全面，1=严重遗漏 |
| **可用性** | 25% | 10=直接使用，1=无法使用 |
| **格式** | 20% | 10=完美符合，1=完全不符 |

## 🤖 自举式优化系统

### 核心：用 AI 优化 AI 提示词

```markdown
# 自举优化提示词

你是一个提示词优化专家。你的任务是优化"提示词优化专家"这个角色的提示词。

## 当前提示词
[粘贴当前的优化提示词]

## 优化目标
1. 提高分析准确性
2. 增加输出可操作性
3. 减少 Token 消耗
4. 适配更多场景

## 约束
- 保持核心功能不变
- 输出格式保持兼容
- 不增加复杂度

## 任务
1. 分析当前提示词的不足
2. 提出优化方案
3. 生成优化版本
4. 自我验证优化效果

请开始优化。
```

### 迭代流程

```
Week 1: 收集 10 个真实优化案例
Week 2: 分析模式，提取共性
Week 3: 更新优化提示词
Week 4: 验证效果，继续迭代
```
