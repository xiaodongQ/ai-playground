# AI Task System V5 Web UI

## 概述

V5 Web UI 提供浏览器端任务管理和实时输出可视化功能。

## 访问方式

```
http://localhost:18792/
```

## 页面结构

| 页面 | URL | 说明 |
|------|-----|------|
| 首页 | `/` | 任务列表 + 统计卡片 |
| 新建任务 | `/new` | 创建任务表单 + 路由预览 |
| 任务详情 | `/task/{id}` | 任务详情 + 实时输出流 |

## 技术实现

### 前端
- **HTML/CSS/JS**: 原生实现，零依赖
- **样式**: 深色主题，现代化 Dashboard 风格
- **实时通信**: WebSocket 复用已有的 `/ws` 端点

### 文件结构
```
v5/web/
├── static/
│   ├── style.css     # 全局样式
│   └── app.js       # 前端逻辑 + API 封装
└── templates/
    ├── index.html   # 任务列表页
    ├── new.html     # 新建任务页
    └── task.html    # 任务详情页
```

### 核心功能

#### 1. 任务列表页 (`/`)
- 统计卡片：pending/running/done/failed 数量
- 任务卡片列表（按创建时间倒序）
- 自动刷新：每 10 秒刷新列表
- WebSocket 监听任务状态变更

#### 2. 新建任务页 (`/new`)
- 表单字段：
  - 任务描述（必填）
  - Agent 选择（自动/Claude/Codex/CodeBuddy）
  - 权限模式（Auto/Bypass/Plan）
  - 超时时间（默认 300 秒）
  - 优先级（Critical/High/Normal/Low）
- 路由预览：输入时自动分析任务类型和推荐 Agent
- 提交后跳转到任务详情

#### 3. 任务详情页 (`/task/{id}`)
- 任务信息：ID、状态、Agent、创建时间、运行时长
- 实时输出：
  - WebSocket 连接实时推送 stdout/stderr
  - 任务状态变更自动更新
  - 完成后获取最终结果
- 状态指示：pending（灰）/ running（蓝脉冲）/ done（绿）/ failed（红）

### API 集成

前端通过 Fetch API 调用后端 REST API：

```javascript
// API 封装
const api = {
  async getTasks(status) { /* GET /tasks */ },
  async getTask(id) { /* GET /tasks/{id} */ },
  async createTask(data) { /* POST /tasks */ },
  async route(prompt) { /* POST /route */ }
};
```

### WebSocket 实时推送

监听的事件类型：
- `task_output` - 实时输出行
- `task_status` - 任务状态变更
- `task_complete` - 任务完成（含结果/错误）

```javascript
ws.on('task_output', (msg) => {
  appendOutput(msg.data.text, 'stdout');
});
```

## 启动方式

```bash
# 启动 API + Web UI
python -m v5.api.app --host 0.0.0.0 --port 18792

# 访问
# Web UI: http://localhost:18792/
# API Docs: http://localhost:18792/docs
```
