## 1. Front Matter 格式

```yaml
---
title: AI 能力集 -- OpenClaw 实战手记
categories: [AI, AI 能力集]
tags: [AI, OpenClaw]
---
```

### 字段说明

| 字段 | 必填 | 说明 | 示例 |
|------|------|------|------|
| `title` | ✅ | 文章标题，**不要有空格** | `AI 能力集 -- OpenClaw 实战手记` |
| `categories` | ✅ | 分类（数组格式，**不要有空格**） | `[AI, AI 能力集]` |
| `tags` | ✅ | 标签（数组格式） | `[AI, OpenClaw]` |

### 不需要的字段

- ❌ `description` - 不需要
- ❌ `date` - 不需要（Jekyll 会自动使用文件名日期）
- ❌ `draft` - 不需要（默认发布）

## 3. 格式要求

### 3.1 标题编号

**使用阿拉伯数字**：
```markdown
## 1. 引言      ✅ 正确
## 一、引言      ❌ 错误

### 1.1 定义    ✅ 正确
### 一、定义    ❌ 错误
```

### 3.2 引言前不要放内容

```markdown
## 1. 引言

直接开始写引言内容。

> ❌ 不要在引言前放引用块或副标题
```

### 3.3 代码块

**必须指定语言**：
````markdown
```go      ✅ 正确
```python  ✅ 正确
```bash   ✅ 正确
```       ❌ 避免
````

**代码块要有说明**：
```markdown
**示例代码**：
```go
func main() {
    // 代码内容
}
```

**输出结果**：
```
Hello World
```
```

### 3.4 图片引用

**格式**：
```markdown
![图片描述](/images/文件名.png)
```

**图片说明**：
```markdown
![架构图](/images/2026-03-09-architecture.png)
*图 1：系统架构图*
```

### 3.5 引用块

```markdown
> 重要提示、注意事项用引用块

> 环境说明：安装在家里单独的主机
```

### 3.6 链接格式

```markdown
[链接文本](完整 URL)
```

**示例**：
```markdown
[OpenClaw 官方文档](https://docs.openclaw.ai)
```

## 5. 发布流程

### 5.1 写作阶段

```
1. Wri·执笔人 撰写博客
   ↓
2. 设置草稿（如需要评审）
   ↓
3. 小黑 - 管家 提交评审
   ↓
4. 用户确认/修改
   ↓
5. 发布（移除草稿标记）
   ↓
6. Git 提交发布
```

### 5.2 评审消息格式

```
🖤 小黑：博客已写好，请老板评审！

【博客预览】
标题：{标题}
路径：{_posts/文件名.md}
GitHub: {预览链接}

【内容摘要】
- {要点 1}
- {要点 2}

请回复：
- "通过" → 立即提交
- "修改 XXX" → 修改后重新评审
```
